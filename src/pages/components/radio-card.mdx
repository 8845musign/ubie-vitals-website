## Feature

- TimeQuestionContentのような、選択した後自動で遷移しないタイプのラジオボタン（選択してもなにもおきないボタン）に使用します
- いわゆる「カード」の形をしています
  - クリック/タップ領域が大きく、比較的操作が容易です
  - その代わり、領域を占有するデメリットも存在します

## Usage

- 同じグループとなる項目には、同じ `name` 属性の値を設定します

### 項目をグルーピングし、タイトルをつける

できる限りタイトルをつけてください。選択肢を列挙しただけでは、何を意味する設定なのかをユーザーは理解でいないかもしれません。

`<fieldset>` 要素を使い[^mdn-fieldset]、`<legend>` を組み合わせることでタイトルを付与できます。

[^mdn-fieldset]: [フィールドセット要素 - HTML: ハイパーテキストマークアップ言語 | MDN](https://developer.mozilla.org/ja/docs/Web/HTML/Element/fieldset)

```jsx
<fieldset>
  <legend>今日の期限</legend>

  <RadioCard name="condition">良い</RadioCard>
  <RadioCard name="condition">悪い</RadioCard>
</fieldset>
```

`<RadioCard>` を内包する選択肢が1つのコンテンツとして独立している場合、`<section>` 要素で囲んでも良いでしょう。その場合、必ず見出し（`<hx>`） を添えてください。

```jsx
<section>
  <h2>本日の体調</h2>

  <RadioCard name="condition">良い</RadioCard>
  <RadioCard name="condition">悪い</RadioCard>
</section>
```

## Keyboard Interaction

- <kbd>Tab</kbd>キーでフォーカスすることができます
- カーソルキーの上下で選択を切り替えることができます
