## Accessibility

`<Button>`や`<LinkButton>`を利用することで、適切な要素がマークアップされます。その場合、ネイティブの要素が持つ機能を継承することができます。

### ネイティブの `<button>` 要素

- フォーカスを受け取り、<kbd>Enter</kbd>キーや<kbd>Space</kbd>キーで押下できます
  - `<div>` と`onClick` でボタンを実装してはいけない[^no-button]理由のひとつです
  - スクリーンリーダーでボタンであることが読み上げられます[^read-button]

[^no-button]: その他の問題として、`<div>` と `onClick` のみの実装ではスクリーンリーダーユーザーに「ボタン」の存在が伝わりません。[［デモ : NVDA］ボタンじゃないボタン 【AccessiブルGoGo! #4 】](https://www.youtube.com/watch?v=XdDvIOMDuCc)
[^read-button]: [［デモ : NVDA］ボタン（button要素）【AccessiブルGoGo! #4 】](https://www.youtube.com/watch?v=vtvu5NBrYkw)
